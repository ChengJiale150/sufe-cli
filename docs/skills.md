# Skill 编写规范

本文档定义了 `skills/<skill-name>/SKILL.md` 的编写标准。所有新增或更新技能时，均应遵循本规范，确保 Agent 能正确理解和执行。

---

## 文件定位

每个技能独立为一个目录，放置在 `skills/` 下：

```text
skills/
└── <skill-name>/
    └── SKILL.md          # 技能定义文件
```

技能目录名使用小写字母和连字符，与 `name` 字段保持一致。

---

## 文件结构

```markdown
---
name: <skill-name>
description: <触发条件描述>
---

[前置依赖提示]

## 核心概念

## 指令速查表

## 工作流
```

---

## Frontmatter

文件开头使用 YAML Frontmatter，包含两个必填字段：

```yaml
---
name: sufe-canvas
description: SUFE Canvas 学习平台业务域。当用户询问课程与作业提交相关的信息时，请务必触发此技能
---
```

**规范：**
- `name`：技能标识符，使用小写字母和连字符，与目录名一致
- `description`：触发条件描述，需明确"当...时，请务必触发此技能"

---

## 前置依赖提示

若当前技能依赖其他技能（如认证、环境检查），应在 Frontmatter 后立即添加提示：

```markdown
**⚠️ 重要提示：在执行本技能中的任何命令前，请务必先参考并触发 `sufe-base` 技能**
```

---

## 核心概念

使用无序列表定义业务术语，每个术语用 `**粗体**` 包裹。

**规范：**
- 面向 Agent 说明业务含义，**禁止**出现技术实现细节（如正则表达式、参数名、算法优先级）
- 支持二级子列表描述细分状态或类型

---

## 指令速查表

使用标准 Markdown 表格列出所有可用命令：

**规范：**
- 表头固定为 `| 指令 | 说明 |`
- 每条说明格式：**动作**：具体功能描述
- 参数标注：
  - `<必填参数>`：尖括号包裹
  - `[可选参数]`：方括号包裹
  - `[--flag <value>]`：选项参数

---

## 工作流

工作流包含多种CLI命令的组合执行, 定义常见业务场景的执行流程。

**规范：**
- 使用描述性名称给出工作流名称
- 使用"当...时，...“格式描述触发场景
- 使用编号列表给出具体执行步骤
  - 条件分支使用缩进子列表
  - 每条步骤应包含具体操作命令

---

## 面向 Agent 的编写原则

1. **语义优先**：说明"是什么"和"能做什么"，而非具体技术细节
2. **路径完备**：提供主路径 + 常见分支（if/else），但不过度枚举异常
3. **约束前置**：危险操作或特殊限制在开头说明
4. **触发明确**：每个工作流必须有清晰的触发条件，让 Agent 知道何时调用

---

## 11. 与代码同步

技能文档必须与代码实现保持一致：

- 新增 CLI 命令时，同步添加至指令速查表
- 修改命令参数时，同步更新文档中的参数说明
- 新增业务场景时，同步补充工作流定义

参考示例：
- `skills/sufe-base/SKILL.md`
- `skills/sufe-canvas/SKILL.md`
