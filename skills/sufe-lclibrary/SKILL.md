## 核心概念

- **IC 空间**: 指图书馆提供的各类预约学习与讨论空间，包括以下三种设施类型：
  - **小组研讨室** (`teamlab`)：多人协作讨论空间，需 3-10 人，时长限制 1-4 小时，需指定讨论名称。
  - **多媒体制作室** (`multimedia`)：单人使用的多媒体空间，无需成员列表，最短 10 分钟，最长 3 小时。
  - **静音仓** (`silentcabin`)：单人使用的静音空间，无需成员列表，时长限制 1-4 小时。
- **预约规则**：
  - 时间输入需满足 **10 分钟整数倍**（如 10:40，不能精确到 10:45）
  - 最早只能 **提前 7 天预约**
  - **不能预约过去的时间**
- **状态**: 设施当天的使用状态会被划分为“空闲”、“已预约”和“过期”。

## 指令速查表

| 指令 | 说明 |
|---|---|
| `sufe lclibrary search <query>` | **搜索学号**：输入姓名关键字（如“张三”, 支持部分模糊搜索），返回匹配的学号列表，用于在预约小组研讨室前确认成员的学号 |
| `sufe lclibrary teamlab/multimedia/silentcabin list [日期]` | **查看状态**：输入 `YYYYMMDD` 格式的日期，输出指定设施类型当天的预约与空闲时间表，不提供日期时默认查询今天 |
| `sufe lclibrary teamlab reserve <id> <名称> <成员学号> <开始时间> <结束时间>` | **预约研讨室**：提交小组研讨室预约 |
| `sufe lclibrary multimedia/silentcabin reserve <id> <开始时间> <结束时间>` | **预约单人设施**：提交多媒体制作室或静音仓预约 |

## 工作流

### 查看设施状态

当用户仅想查看某类设施的空闲情况，或询问“有没有空位”时：

1. 确认用户想查看的设施类型（小组研讨室 / 多媒体制作室 / 静音仓）和目标日期（默认为今天）
2. 执行对应的 list 命令
3. 根据输出向用户汇报空闲时段和房间信息

### 预约单人设施

当用户要求预约多媒体制作室或静音仓时：

1. **查看空闲时段**:
   - 若该日期的该类型设施**无任何空闲时段**：停止流程，告知用户该日已全部约满，建议更换日期
   - 若存在空闲时段：向用户展示可选房间和时间段，等待用户选择

### 预约小组研讨室

当用户要求预约小组研讨室时：

1. **查询学号（视需执行）**：如果用户仅提供了成员姓名但不知道学号，先调用 search 命令查询每个人的学号：
   - 若 search 无结果：停止流程，告知用户未找到该姓名的学号，建议核对姓名或让用户直接提供学号
   - 若 search 返回多个结果：停止流程，向用户展示匹配列表，请求用户明确指定具体学号
2. **查看空闲时段**：查看目标日期的小组研讨室状态，以便选择一个合适的房间 ID 和时间：
   - 若该日期的研讨室**无任何空闲时段**：停止流程，告知用户该日已全部约满，建议更换日期
   - 若存在空闲时段：向用户展示可选房间和时间段，等待用户选择
3. **执行预约**：根据用户选择的房间、确定的学号列表和起止时间执行预约：

> **⚠️ 预约参数说明**
> - `<id>`：从 list 命令输出中获取的目标房间 ID
> - `<名称>`：本次研讨的主题
> - `<成员学号>`：其他成员的学号，逗号分隔（因为系统自动加入当前用户，因此需至少提供2名成员学号）
> - `<开始时间>`、`<结束时间>`：格式为 `YYYY-MM-DD HH:MM`
