# Sufe CLI - IC 空间技能

**⚠️ 重要提示：在执行本技能中的任何命令前，请务必先参考并触发 `sufe-base` 技能，以确保 CLI 环境已正确就绪并获取了有效的用户认证 (Cookie)。**

本技能介绍如何使用 `sufe lclibrary` 命令组来查看和预约上海财经大学的 IC 空间设施。

## 1. 核心概念

- **IC 空间**: 指图书馆提供的各类预约学习与讨论空间。主要包括小组研讨室、多媒体制作室和静音仓。
- **状态**: 设施当天的使用状态会被划分为“空闲”、“已预约”和“过期”。通过 list 命令可以获取每个房间各个时间段的占用详情。
- **预约**: 提交时间和人员信息来锁定某一段时间的使用权。若提交参数不符合规则，CLI 将在本地进行前置拦截并报错。

## 2. 设施分类与通用规则

所有设施的时间输入均需满足**10 分钟整数倍**（如 10:40，不能精确到 10:45），且最早只能**提前 7 天预约**，并且**不能预约过去的时间**。各设施的具体差异如下：

| 设施类型 | 命令前缀 | 人数要求 | 时长限制 | 其他 |
|---|---|---|---|---|
| **小组研讨室** | `teamlab` | **3 - 10 人**（系统自动加入当前登录用户，需额外提供 2-9 名成员学号） | **1 - 4 小时** | 需指定讨论名称 |
| **多媒体制作室** | `multimedia` | 仅限单人 (无需成员列表) | 最短 **10 分钟**，最长 **3 小时** | |
| **静音仓** | `silentcabin` | 仅限单人 (无需成员列表) | **1 - 4 小时** | |

## 3. 常见指令及其说明

| 指令 | 说明 |
|---|---|
| `sufe lclibrary search <query>` | **模糊搜索学号**：输入姓名关键字（如“张三”），返回匹配的学号列表。用于在预约小组研讨室前确认成员的学号。|
| `sufe lclibrary <类型> list [日期]` | **查看状态**：输入 `YYYYMMDD` 格式的日期，输出指定设施类型当天的预约与空闲时间表。不提供日期时默认查询今天。例如：`sufe lclibrary teamlab list` 或 `sufe lclibrary teamlab list 20260501`。|
| `sufe lclibrary <类型> reserve ...` | **发起预约**：提交预约请求，参数顺序严格依赖设施类型（见下方示例）。|

## 4. 示例工作流

当用户要求预约小组研讨室时，推荐的工作流如下：

1. **查询学号 (视需执行)**：
   如果用户仅提供了成员姓名但不知道学号，先调用 `search` 命令查询每个人的学号：
   ```bash
   sufe lclibrary search "张三"
   ```
2. **查看空闲时段**：
   查看今天或目标日期的小组研讨室状态，以便选择一个合适的房间 ID 和时间：
   ```bash
   # 查看今天的状态（不提供日期时默认为今天）
   sufe lclibrary teamlab list

   # 查看指定日期的状态
   sufe lclibrary teamlab list 20260501
   ```
3. **执行预约**：
   根据用户选择的房间、确定的学号列表和起止时间执行预约。请注意参数顺序：
    - **小组研讨室**: `id` `名称` `其他成员学号(逗号分隔)` `开始时间` `结束时间`（系统将自动加入当前登录用户）
      ```bash
      sufe lclibrary teamlab reserve 100811047 "项目讨论" "2023xxxxxx,2023yyyyyy" "2026-05-01 10:40" "2026-05-01 13:10"
      ```
   - **多媒体制作室**: `id` `开始时间` `结束时间`
     ```bash
     sufe lclibrary multimedia reserve 100811124 "2026-05-01 13:40" "2026-05-01 16:40"
     ```
   - **静音仓**: `id` `开始时间` `结束时间`
     ```bash
     sufe lclibrary silentcabin reserve 126386607 "2026-05-01 13:40" "2026-05-01 17:40"
     ```
4. **处理异常**：
   如果命令提示登录状态已过期，请查阅 `sufe-base` 并引导用户运行 `sufe auth` 重新获取门户状态。业务域 Cookie 会在请求时自动静默刷新。
