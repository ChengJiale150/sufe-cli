## 核心概念

- **浏览器环境**：`sufe` 依赖本地浏览器完成自动化操作，首次使用前需确保环境就绪。
- **认证状态**：`sufe` 在本地保存登录凭据，业务调用前自动验证；过期或缺失时需执行 `sufe auth`。

## 指令速查表

| 指令 | 说明 |
|---|---|
| `sufe doctor` | **环境检查**：检查浏览器环境是否就绪，以及登录状态是否可用 |
| `sufe install` | **环境安装**：安装所需的浏览器 |
| `sufe auth` | **用户认证**：执行登录，并将认证状态保存到本地 |
| `sufe me` | **查看当前用户**：显示当前已登录用户的学号、姓名和学院信息 |

> **⚠️ 重要约束**  
> `sufe auth` 可能涉及浏览器弹出或凭据输入，属于用户交互敏感操作。无论何种场景，当检测到认证缺失或过期时，Agent **必须** 明确提示用户自行运行该命令，不得自动触发。

## 工作流

### 工作流一：业务前置检查（由外部 SKILL 触发）

当 `sufe-lclibrary`、`sufe-canvas`、`sufe-score` 等技能准备执行业务命令前，应调用此工作流确保环境就绪。

**路径**：

1. 执行 `sufe doctor`
   - 浏览器缺失 → 执行 `sufe install` → 成功后继续检查认证状态
   - 认证状态缺失/失效 → 尝试自动修复
     - 修复成功 → 继续执行业务命令
     - 修复失败 → 终止业务命令，明确提示用户执行 `sufe auth`
   - 全部正常 → 直接执行业务命令

### 工作流二：直接触发（用户主动询问）

当用户主动询问环境、登录、认证或身份相关问题时：

1. 优先执行 `sufe doctor` 获取当前环境状态，根据输出向用户汇报：
   - 浏览器缺失 → 告知需 `sufe install`
   - 认证缺失/过期 → 告知需 `sufe auth`（明确提示用户自行执行）
   - 全部正常 → 告知已就绪
2. 若用户想确认身份信息 → 执行 `sufe me`
3. 若用户表示已完成 `sufe auth` → 可执行 `sufe doctor` 或 `sufe me` 验证
