# Sufe CLI 基础技能

本技能提供 `sufe` 命令行工具的基础信息与核心工作流。`sufe` CLI 主要用于与上海财经大学 (SUFE) 的网页系统进行自动化交互。

## 1. 核心概念

- **自动化依赖 (Playwright)**: `sufe` 依赖 Playwright 框架进行网页端的模拟操作和交互。它需要一个后台的 Chromium 浏览器作为驱动来完成自动化登录等操作。
- **授权机制 (State)**: 调用校内系统的业务接口前，需要统一身份认证状态。`sufe` CLI 会将 Playwright 登录状态保存到本地，业务域 Cookie 缺失或失效时会尝试通过门户状态静默刷新。

## 2. 常见指令及其说明

| 指令 | 说明 |
|---|---|
| `sufe doctor` | **环境检查**：检查 Playwright Chromium 浏览器是否已安装，以及门户登录状态是否可用。|
| `sufe install` | **环境安装**：安装所需的 Playwright Chromium 浏览器。 |
| `sufe auth` | **用户认证**：执行登录，并将门户状态保存到本地。⚠️ **注意**：您**不需要**在每次使用前都显式运行此命令。本地保存的状态具有一定的有效期。通常情况下，只需在首次使用、或是业务命令抛出“请重新运行 `sufe auth`”的错误提示时，再执行此操作即可。 |
| `sufe me` | **查看当前用户**：显示当前已登录用户的学号、姓名和学院信息。 |
| `sufe -v` | **版本查看**：输出当前 `sufe-cli` 的版本号。 |

## 3. 示例工作流

当您在一台全新的设备上配置或首次使用 `sufe` 时，推荐的工作流如下：

1. **判断环境是否就绪**：
   运行环境检查命令。
   ```bash
   sufe doctor
   ```
   该命令会依次检查：
   - Playwright Chromium 浏览器是否已安装
   - 门户登录状态是否存在且可用
2. **处理依赖缺失 (视需执行)**：
   如果 `doctor` 命令报告找不到 Chromium 浏览器（显示红色的 ❌ 错误提示），则需要运行安装命令：
   ```bash
   sufe install
   ```
3. **完成首次认证**：
   环境就绪后，执行认证以获取门户状态：
   ```bash
   sufe auth
   ```

4. **验证登录信息 (视需执行)**：
   如需确认当前登录的用户身份，可以运行：
   ```bash
   sufe me
   ```

完成上述工作流后，您就可以参考其他相关技能（如 `sufe-lclibrary`）直接调用相应的业务命令了。
